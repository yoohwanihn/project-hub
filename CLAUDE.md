# Project Context: 차세대 스마트 협업 관리 시스템

## 역할 정의 (Claude)
너는 이 프로젝트의 **시니어 풀스택 개발자**로 동작한다.  
모든 코드는 확장성, 유지보수성, 실무 기준을 우선한다.

---

## 프로젝트 개요
React + Node.js 기반 통합 프로젝트 관리 플랫폼.

목표:
- 업무 이력 통합 및 지식 자산화
- 실시간 협업 및 업무 투명성 확보
- 한국형 UX 최적화

개발 기간: 12주

---

## 핵심 기능 (우선순위 포함)

### P0 (핵심 기능)
- 프로젝트 / 업무(Task) 관리
- 칸반 보드 (Drag & Drop)
- 간트 차트 (일정 시각화)
- 개인화 대시보드

### P1 (중요 기능)
- 프로젝트 위키 (버전 관리)
- 실시간 타임라인 (이벤트 로그)

### P2 (부가 기능)
- 시간 추적 (공수 관리)
- 파일 보관 및 검색
- 공지사항 / 투표

---

## 기술 스택

### Frontend
- React (Hooks 기반)
- 상태 관리: Zustand (기본), 필요 시 Context 병행
- UI: 반응형 설계

### Backend
- Node.js (NestJS 우선)
- RESTful API 설계

### Database
- PostgreSQL
- 관계형 데이터 모델 (Task Dependency 필수 지원)

### 인증 / 보안
- JWT 기반 인증
- bcrypt 비밀번호 암호화

## 데이터 설계 원칙

- 모든 엔티티는 UUID 사용
- Task는 다음 관계를 지원해야 한다:
  - parent-child
  - dependency (선후행)
- soft delete 사용 (deleted_at)
- audit 필드 필수:
  - created_at
  - updated_at

---

## 성능 전략

- 리스트 조회는 기본 pagination 적용
- 무한 스크롤 지원
- 불필요한 리렌더링 방지 (memoization 필수)
- 대용량 데이터는 서버에서 필터링
